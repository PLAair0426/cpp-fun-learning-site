# C++ 趣味学习网站 Execution Plan

> 目标：在保持单仓库协作的前提下，明确 `frontend`、`backend/api`、`backend/worker` 与部署目录的职责边界。

## 1. 项目目标

- 保持单仓库，降低当前迭代阶段的联调成本
- 明确前端、API、Worker 的职责，减少耦合
- 为后续接入判题、队列、SSE 和后台运营能力预留空间

## 2. 执行阶段

### 阶段一：结构整理

- 统一仓库目录
- 清理无关文件
- 固化 frontend / backend 边界

### 阶段二：能力拆分

- 前端负责页面、交互、认证入口与后台界面
- API 负责内容接口、用户、进度与提交数据
- Worker 负责异步任务与判题扩展

### 阶段三：部署验证

- Docker Compose 启动依赖服务
- 验证 API 与 Frontend 联通
- 为后续 worker 扩容预留配置

## 3. 风险

- 文档与目录调整时容易遗漏引用
- 编码问题会影响模板与说明文档的可维护性
- 判题与异步能力落地后会提高运维复杂度

## 4. 验证方式

- 前端构建通过
- API 首页聚合接口可访问
- 核心页面无 500 且主要文档可读
