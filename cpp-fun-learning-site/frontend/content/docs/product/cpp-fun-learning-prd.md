# C++ 趣味学习网站 PRD

## 1. 文档信息

- 项目名称：`C++ 趣味学习网站`
- 当前版本：`v0.2`
- 相关资料：`frontend/content/docs/product/course-map-from-docx.md`

## 2. 产品背景

项目目标是把传统 C++ 课件、示例代码和练习题整理为一个可持续运营的学习平台，让用户能在同一站点内完成学习、练习、提交与进度追踪。

## 3. 产品目标

- 为 C++ 初学者提供从基础语法到综合练习的路线化学习体验
- 将课程、题库、进度与用户记录统一到同一平台
- 通过更轻松舒适的 UI 降低入门压力
- 为后续判题、排行榜、运营和管理后台预留扩展能力

## 4. 目标用户

- C++ 零基础学习者
- 需要刷题和巩固知识点的学生
- 想按路径系统补课的自学用户
- 需要维护课程内容的管理员

## 5. 核心场景

- 新用户注册并登录，建立个人学习记录
- 用户按学习路径逐课学习，并完成配套练习
- 用户在题目页提交代码并查看结果
- 管理员查看用户数据并持续补充课程内容

## 6. 核心功能

- 首页与课程目录展示
- 学习路径与章节页面
- 题库页、题目详情页、代码运行面板
- 用户注册、登录、个人记录隔离
- 管理后台：用户、内容、数据概览

## 7. 非目标

- 当前版本不追求完整 LMS 商业化能力
- 当前版本不覆盖移动端原生应用
- 当前版本不把判题服务拆成独立仓库

## 8. 关键指标

- 注册转化率
- 首课完成率
- 每周活跃学习用户
- 题目提交次数与通过率
- 用户连续学习天数

## 9. 约束与风险

- 原始资料来自 DOCX，内容结构化需要持续人工校对
- 中文文档与脚本处理时需要避免编码损坏
- 判题与异步任务会提高部署复杂度

## 10. 后续规划

- 接入真实判题能力
- 扩展更多学习路径与模块
- 完善管理台的内容编辑与审核能力
- 补全数据分析与运营工具
