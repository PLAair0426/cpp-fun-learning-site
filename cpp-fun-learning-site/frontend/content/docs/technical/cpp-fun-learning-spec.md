# C++ 趣味学习网站 Spec

## 1. 目标与边界

本规格说明围绕当前学习平台的 Web、API、用户体系、内容体系与后台能力展开。

## 2. 系统模块

- Web 前台：首页、路径页、题库页、题目页、认证页
- Admin 后台：用户与内容管理、数据概览
- API：内容数据、用户认证、进度与提交
- Worker：异步任务、判题与后续事件处理
- 数据层：PostgreSQL + Redis

## 3. 角色与权限

- 游客：浏览公开页面与课程概览
- 普通用户：学习、提交、查看自己的记录
- 管理员：管理用户、内容与后台数据

## 4. 内容模型

- Path：学习路径
- Module：路径下的模块
- Lesson：课程章节
- Problem：题目
- Submission：提交记录
- UserProgress：用户进度与推荐动作

## 5. 关键流程

### 5.1 用户访问流程

1. 用户进入首页
2. 浏览路径、课程与题目预览
3. 注册或登录后建立个人记录
4. 进入路径页或题库页继续学习

### 5.2 学习流程

1. 打开路径
2. 阅读课程内容
3. 完成配套练习
4. 更新进度、经验值和推荐动作

### 5.3 管理流程

1. 管理员进入后台
2. 查看用户与内容概览
3. 新增或维护课程与题目
4. 查看数据变化并继续迭代内容

## 6. 接口要求

API 至少应覆盖：

- 首页聚合数据
- 路径列表与详情
- 题库列表与题目详情
- 用户认证
- 进度总览
- 管理端概览与用户数据

## 7. 非功能要求

- 前端页面需保持可读性与舒适配色
- 接口响应需稳定，避免首页与核心页面 500
- 文档与模板统一采用 UTF-8
- 容器化启动应支持本地快速验证

## 8. 风险与依赖

- 原始资料结构化过程中可能存在内容缺口
- 判题外部服务接入会增加复杂度
- 文档编码处理不当会导致说明材料再次损坏

## 9. 后续扩展

- 接入 Judge0 或自建判题
- 引入队列、SSE 与排行榜
- 扩展更多课程模块与运营能力
